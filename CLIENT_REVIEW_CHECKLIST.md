## 🔍 검토 방법

### 1. 준비사항
- [ ] 개발 서버 URL 확인: `http://localhost:3010` (로컬) 또는 배포 URL
- [ ] 테스트용 계정 정보 확인
  - 일반 사용자 계정
  - 호스트 계정
  - 관리자 계정 (전화번호: 010-5470-8008, 010-899-41584)
- [ ] 다양한 기기에서 테스트 (PC, 태블릿, 모바일)
- [ ] 주요 브라우저에서 테스트 (Chrome, Safari, Firefox)

### 2. 검토 순서
1. **필수 기능** 먼저 테스트
2. **사용자 시나리오** 기반 테스트 (실제 사용 흐름대로)
3. **예외 상황** 테스트 (오류 처리, 엣지 케이스)
4. **UI/UX** 검토 (디자인, 사용성)
5. **성능** 확인 (로딩 속도, 반응성)

## ✅ 기능별 체크리스트

### 1. 브랜딩 (최근 작업)
- [ ] 모든 "ChonCance" → "VINTEE"로 변경 확인
- [ ] 로고 이미지 정상 표시 (/vintee-logo.png)
- [ ] 메타 태그 (페이지 제목, description) 확인
- [ ] Footer 브랜드명 확인

### 2. 랜딩 페이지
- [ ] 히어로 섹션 정상 표시
- [ ] 검색바 동작 확인
- [ ] 이용방법 섹션 표시
- [ ] 추천 숙소 섹션 표시
- [ ] 반응형 디자인 (모바일/태블릿/PC)

### 3. 사용자 인증 (Clerk)
- [ ] 회원가입 동작
- [ ] 로그인/로그아웃 동작
- [ ] 소셜 로그인 (Google 등)
- [ ] 사용자 프로필 표시

### 4. 숙소 탐색 (/explore)
- [ ] 숙소 목록 정상 표시
- [ ] 태그 필터링 동작 (VIEW, ACTIVITY, FACILITY, VIBE)
- [ ] 검색 기능 동작
- [ ] 숙소 카드 클릭 시 상세 페이지 이동
- [ ] 이미지 정상 로드

### 5. 숙소 상세 페이지 (/property/[id])
- [ ] 숙소 정보 정확히 표시
- [ ] 이미지 갤러리 동작
- [ ] 호스트 정보 표시
- [ ] 태그 표시
- [ ] 관련 숙소 추천

### 6. 호스트 기능
#### 6.1 호스트 등록 (/become-a-host)
- [ ] 호스트 신청 폼 동작
- [ ] 필수 항목 유효성 검증
- [ ] 제출 후 승인 대기 상태 확인

#### 6.2 숙소 등록 (/host/properties/new)
- [ ] 이미지 업로드 기능 (파일 업로드)
- [ ] Gemini AI 이미지 최적화 동작
- [ ] 썸네일 선택 기능
- [ ] 숙소 정보 입력 폼
- [ ] 태그 선택 기능
- [ ] 등록 후 "검토 중" 상태 확인

#### 6.3 내 숙소 관리 (/host/properties)
- [ ] 등록한 숙소 목록 표시
- [ ] 숙소별 상태 표시 (APPROVED/PENDING)
- [ ] 숙소 수정 기능
- [ ] 숙소 이미지 정상 표시

### 7. 관리자 기능
#### 7.1 QR 로그인 (/admin/login)
- [ ] QR 코드 생성 및 표시
- [ ] QR 코드 만료 시간 카운트다운 (5분)
- [ ] 모바일에서 QR 스캔 후 인증 페이지 열림

- [ ] 인증 완료 후 PC 자동 로그인
- [ ] 국제 번호 형식 지원 (+82, +81, +84)

#### 7.2 숙소 승인 (/admin/properties/pending)
- [ ] 대기 중인 숙소 목록 표시
- [ ] 숙소 상세 정보 확인
- [ ] 승인/거부 기능
- [ ] 상태 변경 후 목록 업데이트

### 8. 예약 시스템

- [ ] 가용성 확인
- [ ] 예약 정보 입력

- [ ] 예약 확인 페이지

### 9. 이메일 알림

## 💬 개발자와의 커뮤니케이션

### 1. 커뮤니케이션 채널
#### 📧 이메일
- 공식 문서나 중요한 결정사항
- 개발 완료 보고
- 계약 관련 사항

#### 💬 채팅/메신저
- 일상적인 질문과 답변
- 빠른 피드백 필요 시
- 진행 상황 확인

#### 🐛 이슈 트래커 (GitHub Issues 권장)
- 버그 리포트
- 기능 요청
- 개선 제안

### 2. 효과적인 피드백 작성법

#### ✅ 좋은 예시:
```
**문제**: 숙소 등록 페이지에서 이미지 업로드 후 썸네일이 표시되지 않습니다.

**재현 방법**:
1. /host/properties/new 접속
2. 이미지 5개 업로드
3. 업로드 완료 후 썸네일 선택 UI 표시 안됨

**예상 동작**: 업로드 후 썸네일 선택 그리드가 표시되어야 함

**환경**:
- 브라우저: Chrome 120
- OS: Mac OS 14.1
- 스크린샷: [첨부]
```

#### ❌ 나쁜 예시:
```
이미지가 안 떠요. 고쳐주세요.
```

### 3. 우선순위 표시
피드백에 우선순위를 명시하면 개발자가 작업 순서를 정하기 쉽습니다:

- **🔴 긴급 (Critical)**: 서비스 사용 불가 (로그인 안됨, 결제 오류 등)
- **🟠 높음 (High)**: 핵심 기능 문제 (숙소 검색 오류 등)
- **🟡 중간 (Medium)**: 부가 기능 문제 (이미지 로딩 느림 등)
- **🟢 낮음 (Low)**: UI 개선, 오타 수정 등

## ✅ 승인 프로세스

### 1. 단계별 승인

#### Phase 1: 기능 검토
1. 모든 체크리스트 항목 테스트
2. 발견한 이슈 리스트 작성
3. 개발자에게 피드백 전달

#### Phase 2: 수정 확인
1. 개발자가 수정 완료 보고
2. 수정 사항 재검증
3. 추가 이슈 발견 시 반복

#### Phase 3: 최종 승인
1. 모든 필수 기능 정상 동작 확인
2. 치명적인 버그 없음 확인
3. 최종 승인서 작성

### 2. 승인 기준

#### ✅ 승인 가능 조건
- [ ] 모든 필수 기능 정상 동작
- [ ] 치명적인 버그 없음 (🔴, 🟠 우선순위)
- [ ] 주요 브라우저에서 정상 동작
- [ ] 모바일 반응형 정상

#### ⚠️ 조건부 승인
- [ ] 경미한 버그 존재 (🟡, 🟢 우선순위)
- [ ] 개선 사항 있으나 사용 가능
- [ ] 추후 업데이트 일정 합의

#### ❌ 승인 불가
- [ ] 핵심 기능 동작 불가
- [ ] 치명적인 보안 문제
- [ ] 데이터 손실 위험
