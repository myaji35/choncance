## 1. 배경 및 목적 (Background & Purpose)

### 1.1 왜 GEO인가?

한국 검색 트렌드가 급격히 변화하고 있습니다:

| 검색 채널 | 2024 | 2026 예상 | 비고 |
|---------|------|--------|-----|
| Google 전통 검색 | 60% | 40% | AI Overview 확대 |
| ChatGPT/Perplexity | 5% | 20% | 여행 추천 쿼리 급증 |
| 네이버 AI 검색 | 10% | 25% | 클로바X 통합 |
| 기타 | 25% | 15% | |

**"강원도 감성 펜션 추천"** 을 ChatGPT에게 물어보면 → VINTEE에 등록된 숙소가 나와야 합니다.

### 1.2 VINTEE GEO의 핵심 목표

```
사용자: "아이와 함께 논뷰 보는 조용한 펜션 어디 좋아요?"
AI 답변: "강원도 홍천의 [논뷰 펜션 이름]은 VINTEE에서 4.8점을 받은 곳으로,
         실제 투숙객들이 '아이와 함께 논밭을 바라보며 힐링했다'고 리뷰했습니다."
```

## 2. 문제 분석 (Problem Analysis)

### 2.1 현재 상태

| 항목 | 현황 |
|-----|-----|
| 구조화 데이터 | 없음 (JSON-LD 미적용) |
| 숙소 상세 페이지 콘텐츠 | 단순 스펙 나열 |
| 리뷰 데이터 노출 | 검색엔진 비가시화 |
| 호스트 스토리 | 텍스트만 있고 구조화 없음 |
| 사이트맵 | 기본 설정만 |
| AI 검색 캐싱 | 전무 |

### 2.2 LLM이 숙소를 추천하는 원리

```
[LLM 추천 메커니즘]

1. Pre-training 데이터 학습
   → 웹에서 "XX 펜션은 Y지역에서 유명하다" 같은 문장이 많을수록 학습됨

2. RAG (Retrieval Augmented Generation)
   → ChatGPT Plus, Perplexity는 실시간 웹 검색 후 답변 생성
   → 검색 결과 상위 노출이 핵심

3. 구조화 데이터 파싱
   → JSON-LD Schema.org 마크업이 있으면 AI가 직접 파싱
   → LodgingBusiness, Review, AggregateRating 스키마

4. 신뢰 신호 (Trust Signals)
   → 다수의 외부 사이트가 동일 숙소를 언급할수록 신뢰도 상승
   → 리뷰 플랫폼 일관성 (네이버, 카카오, VINTEE 점수 일치)
```

## 3. GEO 전략 프레임워크 (Strategy Framework)

### 3.1 3-Layer GEO 모델

```
┌─────────────────────────────────────────────────┐
│  Layer 3: 외부 신호 증폭 (Off-Page GEO)          │
│  블로그 배포, 소셜 언급, 언론 노출, 인플루언서    │
├─────────────────────────────────────────────────┤
│  Layer 2: 구조화 콘텐츠 (On-Page GEO)           │
│  JSON-LD, FAQ, 리뷰 구조화, 호스트 스토리 최적화  │
├─────────────────────────────────────────────────┤
│  Layer 1: 기술적 기반 (Technical GEO)            │
│  LLM.txt, 사이트맵, 메타데이터, 페이지 속도       │
└─────────────────────────────────────────────────┘
```

### 3.2 VINTEE 특화 GEO 전략 (5가지 핵심)

#### 전략 1: LLM 친화적 콘텐츠 구조화
- 숙소 상세 페이지에 **Schema.org LodgingBusiness** JSON-LD 적용
- 리뷰에 **Review + AggregateRating** 마크업
- FAQ 섹션 추가 (자주 묻는 질문 = LLM이 답변할 소재)

#### 전략 2: "AI 인용 가능한" 문장 설계
- 호스트 스토리를 **인용 가능한 단문**으로 재구성
- 예: "이 숙소는 강원도 홍천에 위치한 전통 논뷰 펜션으로, 2대째 운영 중인 가족 농장입니다."
- 구체적 수치 포함: "150평 규모의 논을 직접 경작하며..."

#### 전략 3: 테마 태그 연계 FAQ 자동 생성
- 각 태그별 FAQ 자동 생성: "#논뷰맛집" → "논뷰가 보이는 펜션의 특징은?"
- 숙소별 맞춤 FAQ: 호스트 인터뷰 → FAQ 변환 파이프라인

#### 전략 4: VINTEE Score를 신뢰 신호로 활용
- "VINTEE Score 4.8 / 5.0 — 전국 상위 5%" 문구 구조화
- 다중 플랫폼 리뷰 통합 점수를 **공신력 있는 제3자 평가**로 포지셔닝

#### 전략 5: llms.txt 파일 구현
- `/llms.txt` — AI 크롤러를 위한 사이트 요약 파일 (신규 표준)
- 각 숙소별 `/property/[id]/llms.txt` 생성 고려

## 4. 구현 범위 (Scope)

### 4.1 Sprint 1 — 기술 기반 (2주)
- [ ] 숙소 상세 페이지 JSON-LD 구조화 (LodgingBusiness + Review)
- [ ] `/llms.txt` 파일 생성 (사이트 전체 소개)
- [ ] 동적 사이트맵 개선 (숙소 페이지 우선순위 1.0)
- [ ] Open Graph + Twitter Card 완성

### 4.2 Sprint 2 — 콘텐츠 최적화 (2주)
- [ ] 숙소 상세 FAQ 섹션 컴포넌트 개발
- [ ] 호스트 스토리 AI 최적화 가이드라인 + 입력 템플릿 개선
- [ ] 태그별 랜딩 페이지 SEO/GEO 강화
- [ ] 리뷰 페이지네이션 → AI 가시화

### 4.3 Sprint 3 — 외부 신호 (지속)
- [ ] 블로그 자동 배포 파이프라인 (숙소 소개글 자동 생성)
- [ ] 지역 커뮤니티 / 여행 카페 노출 전략
- [ ] 언론사 보도자료 배포 (VINTEE Score 발표)
