## 1. 개요 (Overview)

### 1.1 목표
외부 플랫폼(네이버, 구글, 여기어때, 에어비앤비 등)에서 펜션의 **기본 정보 + 평점/리뷰 데이터**를 수집하여,
VINTEE 내부 **VINTEE Score(별점)** 산정에 활용하고, 수집 현황을 어드민 대시보드에서 관리한다.

### 1.2 비즈니스 배경
- VINTEE에 신규 등록되는 펜션은 자체 리뷰가 없어 신뢰도 지표 부재
- 외부 플랫폼의 기존 평점/리뷰를 활용하면 초기 신뢰도 확보 가능
- 수집 데이터로 VINTEE Score를 객관적으로 산정하여 게스트 의사결정 지원

### 1.3 핵심 가치
- 신규 호스트도 첫날부터 객관적인 점수 보유
- 다중 소스 평균으로 편향 없는 품질 지표
- 수집 자동화로 운영 부담 최소화

## 3. VINTEE Score 산정 방식

외부 수집 데이터를 기반으로 VINTEE Score(1.0~5.0) 자동 계산:

```
VINTEE Score =
  (네이버 평점 × 0.30) +
  (구글 평점  × 0.30) +
  (리뷰 볼륨 점수 × 0.20) +   // 리뷰 100개 이상 = 5점
  (여기어때 평점 × 0.10) +
  (에어비앤비 평점 × 0.10)
```

- 소스가 없는 항목은 나머지 가중치로 재분배
- 최소 1개 소스 이상 있을 때만 Score 표시
- 수동 오버라이드 가능 (어드민)

## 4. 수집 프로세스 (Collection Process)

```
[호스트 숙소 등록]
       ↓
[자동 수집 트리거] — 등록 즉시 OR 어드민 수동 실행
       ↓
[수집 파이프라인]
  ├── 네이버 플레이스 검색 (숙소명 + 주소)
  ├── 구글 맵스 검색 (Google Places API)
  ├── 여기어때 검색 (크롤링)
  └── 에어비앤비 검색 (크롤링)
       ↓
[데이터 정규화] — 소스별 평점 → 5점 만점 통일
       ↓
[VINTEE Score 계산]
       ↓
[DB 저장] — PropertyExternalData 모델
       ↓
[어드민 대시보드 반영]
```

## 6. 데이터 모델 (Prisma Schema 추가)

```prisma
// 외부 소스별 수집 데이터
model PropertyExternalData {
  id           String   @id @default(cuid())
  propertyId   String
  property     Property @relation(fields: [propertyId], references: [id])

  source       String   // "naver" | "google" | "yeogieo" | "airbnb" | "kakao"
  externalId   String?  // 해당 플랫폼의 ID
  externalUrl  String?  // 해당 플랫폼 URL

  rating       Float?   // 5점 만점 정규화
  reviewCount  Int?
  priceMin     Int?     // 최저가 (원)

  rawData      Json?    // 원본 데이터 전체
  status       String   @default("pending") // pending | success | failed
  errorMessage String?

  collectedAt  DateTime?
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt

  @@unique([propertyId, source])
}

// VINTEE Score (계산된 최종 점수)
model VinteeScore {
  id            String   @id @default(cuid())
  propertyId    String   @unique
  property      Property @relation(fields: [propertyId], references: [id])

  score         Float    // 1.0 ~ 5.0
  scoreBreakdown Json?   // 소스별 기여 내역
  isManualOverride Boolean @default(false)
  overrideNote  String?

  calculatedAt  DateTime @default(now())
  updatedAt     DateTime @updatedAt
}
```

## 7. 기술 구현 전략

### 7.1 API 활용 (공식)
- **Google Places API**: 유료($0.017/요청), 가장 안정적
- **Kakao Local API**: 무료, 한국 특화

### 7.2 크롤링 (법적 주의사항 적용)
- CLAUDE.md 크롤링 규칙 준수: `source: "collected"` 명시
- User-Agent 로테이션, 요청 간격 준수 (1~3초)
- robots.txt 확인 후 허용 범위 내 수집
- 수집 목적: 내부 품질 지표 산정용 (재판매 금지)

### 7.3 수집 주기
- 최초 수집: 숙소 등록 즉시 (비동기)
- 정기 업데이트: 주 1회 자동 (cron job)
- 수동 수집: 어드민에서 언제든 실행 가능
