## 1. 배경 및 목표

### 1.1 이전 스프린트 성과 (sprint-통합)

- **93% Match Rate** 달성 (65% → 93%, 5회 반복)
- Epic 3~6 핵심 기능 구현 완료
- 리뷰 시스템, 호스트/숙소 관리자 승인, 태그 관리 완성

### 1.2 남은 UX 痛點 (Pain Points)

sprint-통합 보고서에서 도출된 **70~89% 완성도 구간** 기능들이 타깃:

| 문제 | Story | 현재 완성도 | 사용자 영향 |
|------|-------|-----------|-----------|
| 멀티스텝 숙소 등록 폼 미완성 | 3.2 | 70% | 호스트 온보딩 이탈 ↑ |
| 숙소 상세에 리뷰 미노출 | 5.2 | 80% | 신뢰도 ↓, 전환율 ↓ |
| 예약 시 가격 명세 불명확 | 4.3 | 70% | 결제 포기 ↑ |
| 모바일 반응형 미흡 | 전반 | 60% | 모바일 이탈율 ↑ |
| 빈 상태(Empty State) 없음 | 전반 | 40% | UX 미완성 느낌 |

### 1.3 이번 스프린트 목표

```
UX 완성도: 현재 평균 70% → 목표 90%+

핵심 사용자 여정 3가지 완성:
1. 호스트 여정: 숙소 등록 → 관리 (멀티스텝 폼 완성)
2. 게스트 여정: 탐색 → 상세보기 → 예약 (리뷰 + 가격 명세)
3. 공통 UX: 빈 상태, 로딩, 에러 처리
```

## 2. 스코프 정의

### 2.1 IN SCOPE (이번 스프린트)

#### P0: 반드시 완성 (사용자 여정 블로커)

**[UX-01] 숙소 상세 페이지 리뷰 섹션 표시** ← Story 5.2
- 숙소 상세(`/property/[id]`)에 리뷰 목록 렌더링
- 평점 요약 (별점 평균, 분포 바차트)
- 리뷰 카드 (사용자명, 날짜, 별점, 내용, 호스트 답변)
- 리뷰 없을 때 Empty State
- **이유**: 리뷰 시스템을 구현했으나 실제 상세 페이지에 미통합 (핵심 신뢰 요소)

**[UX-02] 예약 가격 명세 개선** ← Story 4.3
- 평일/주말 요금 분리 계산 및 표시
- "숙박비 × N박 = 금액" 형태 명세 카드
- 서비스 수수료(10%) 명시
- 체크인/아웃 변경 시 실시간 업데이트
- **이유**: 현재 단순 합계만 표시 → 사용자 불신 → 결제 포기

**[UX-03] 멀티스텝 숙소 등록 폼** ← Story 3.2
- 8단계 → **5단계로 단순화** (MVP 최적화)
  - Step 1: 기본 정보 (이름, 유형, 지역, 주소)
  - Step 2: 시설/편의 (체크박스 그리드)
  - Step 3: 이미지 (기존 업로드 컴포넌트 재활용)
  - Step 4: 스토리 & 태그 (텍스트 + 태그 선택)
  - Step 5: 가격 & 제출 (평일/주말 요금 + 검토 요청)
- Progress bar (Step N / 5)
- localStorage 임시 저장 (이탈 후 복귀 시 복원)
- **이유**: 현재 단일 긴 폼 → 호스트 이탈 최대 구간

#### P1: 완성하면 크게 좋아지는 것

**[UX-04] Empty State 컴포넌트 통합** ← 전반
- 예약 없을 때, 리뷰 없을 때, 숙소 없을 때 공통 Empty State UI
- 액션 버튼 포함 ("첫 숙소 등록하기", "예약 내역 보기")
- Lottie 또는 SVG 일러스트 (농촌 테마)

**[UX-05] 탐색 페이지 무한 스크롤 / 페이지네이션** ← Story 1.1
- 현재: 전체 숙소 한 번에 로드 (성능 이슈)
- 개선: Cursor 기반 페이지네이션 (12개 단위)
- Skeleton 로딩 카드

#### P2: 여유 시 추가

**[UX-06] 모바일 헤더 & 하단 내비게이션**
- 모바일 전용 Bottom Navigation (홈/탐색/예약/프로필)
- 모바일 SiteHeader 간소화

## 4. 기술 접근 전략

### 4.1 재사용 우선 원칙

sprint-통합에서 이미 구현된 자산 활용:
- `ReviewForm`, `ReviewCard` 컴포넌트 → property 상세에 통합
- `image-upload.tsx` → 멀티스텝 Step 3에 그대로 적용
- `BookingWidget` 가격 계산 로직 → PriceBreakdown으로 분리

### 4.2 컴포넌트 설계 원칙

```
새로 만드는 컴포넌트:
- EmptyState (범용, props로 아이콘/제목/설명/CTA 주입)
- PriceBreakdown (BookingWidget에서 분리)
- MultiStepForm (5단계 컨테이너)
- RatingSummary (평균 별점 + 분포)
- ReviewList (숙소 상세용)
```

### 4.3 데이터 플로우

```
[UX-01] 리뷰 표시:
  property/[id]/page.tsx
    → GET /api/reviews?propertyId={id}
    → ReviewList → ReviewCard (기존 컴포넌트 활용)

[UX-02] 가격 명세:
  BookingWidget (날짜 상태 보유)
    → PriceBreakdown (날짜 + property 요금 받아 계산)
    → 평일/주말 분리 표시

[UX-03] 멀티스텝 폼:
  host/properties/new/page.tsx
    → MultiStepForm (5 steps)
    → localStorage 드래프트 저장
    → POST /api/host/properties (기존 API 재활용)
```
