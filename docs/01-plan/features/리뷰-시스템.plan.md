---

## 1. 개요 (Overview)

VINTEE 플랫폼의 리뷰 시스템을 구현한다.
체크아웃 완료된 게스트가 숙소 리뷰를 작성하고, 호스트가 답글을 달 수 있는 기능이다.
신뢰 기반 플랫폼 구축을 위한 핵심 요소이며, 향후 SNS 공유 크레딧 보상 시스템과 연계된다.

## 5. 기술 설계 방향 (Technical Direction)

### DB 스키마
> ✅ Review 모델 이미 정의됨. 마이그레이션 불필요.
```
Review
  - bookingId (unique) → 예약 1건당 리뷰 1개
  - userId → 게스트
  - propertyId → 숙소
  - rating: Int (1-5)
  - content: String
  - hostReply: String? → 호스트 답글 (별도 모델 대신 인라인)
  - repliedAt: DateTime?
```

### API 설계
```
POST   /api/reviews                    → 리뷰 작성
GET    /api/properties/[id]/reviews    → 숙소 리뷰 목록 (페이지네이션)
POST   /api/reviews/[id]/reply         → 호스트 답글
GET    /api/host/reviews               → 호스트 리뷰 관리
```

### 컴포넌트 설계
```
src/components/review/
  ├── StarRating.tsx         (별점 선택/표시)
  ├── ReviewCard.tsx         (리뷰 카드 표시)
  ├── ReviewList.tsx         (리뷰 목록 + 페이지네이션)
  └── HostResponseForm.tsx   (호스트 답글 폼)

src/app/
  ├── bookings/[id]/review/page.tsx    (리뷰 작성 페이지)
  └── host/reviews/page.tsx            (호스트 리뷰 관리)
```

### 주요 비즈니스 로직
- **리뷰 작성 조건**: `booking.status === COMPLETED` AND `booking.checkOut < now()` AND `review === null`
- **이름 마스킹**: `홍길동` → `홍*동` (개인정보 보호)
- **평균 별점**: Prisma `aggregate._avg.rating`
- **중복 방지**: `bookingId @unique` (DB 레벨)

## 6. 구현 순서 (Implementation Order)

```

```
