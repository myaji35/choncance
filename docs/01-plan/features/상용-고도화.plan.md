## 2. 고도화 로드맵

### Sprint 1: 핵심 플로우 완성 (P0)

> 목표: 게스트가 숙소를 찾고 예약하고 리뷰를 남기는 E2E 플로우 완성

#### 2.1 숙소 등록 폼 페이지
```
Path: /host/properties/new
구현:
- 숙소명, 설명, 지역, 주소, 가격, 최대 인원 입력
- react-hook-form + Zod 검증
- 임시저장(draft) / 바로 공개(active) 선택
- 등록 후 호스트 대시보드로 리다이렉트
```

#### 2.2 숙소 상세 예약 폼
```
Path: /property/[id] 내 예약 섹션
구현:
- 날짜 선택 (체크인/체크아웃) — input[type=date] 사용
- 인원 선택 (1~maxGuests)
- 가격 자동 계산 (박 수 × 1박 가격)
- 호스트에게 메시지 (선택)
- 로그인 미인증 시 "로그인 후 예약 가능" 안내
- POST /api/bookings 호출
```

#### 2.3 호스트 예약 관리
```
Path: /host/bookings
API: PATCH /api/bookings/[id]/status
구현:
- 대기중(PENDING) 예약 승인/거절 버튼
- 확정(CONFIRMED) 예약 → 완료 처리 버튼
- 상태별 필터 (전체/대기/확정/완료/취소)
```

#### 2.4 에러 바운더리 + Not Found
```
Files:
- src/app/error.tsx (글로벌 에러 바운더리)
- src/app/not-found.tsx (404 페이지)
- src/app/bookings/[id]/not-found.tsx
- src/app/property/[id]/not-found.tsx
```

#### 2.5 미들웨어 보호 라우트
```
File: src/middleware.ts
보호 대상:
- /bookings/* → 로그인 필수
- /host/* → HOST 역할 필수
- /api/bookings → 로그인 필수
- /api/properties (POST) → HOST 역할 필수
```

#### 2.6 시드 데이터 확충
```
데모 계정:
- host@vintee.kr / password123 (호스트)
- guest@vintee.kr / password123 (게스트)
- admin@vintee.kr / password123 (관리자)

데모 숙소 5개:
- 논뷰 한옥 펜션 (충남 아산, 12만원)
- 산골 오두막 (강원 평창, 8만원)
- 바다 근처 민박 (경남 통영, 10만원)
- 감귤 농장 스테이 (제주 서귀포, 15만원)
- 대나무 숲 글램핑 (전남 담양, 9만원)

데모 예약 + 리뷰: 각 숙소 1~2건
```

### Sprint 2: UX 완성도 (P1)

#### 2.7 숙소 수정/삭제
```
API: PUT/DELETE /api/properties/[id]
UI: /host/properties/[id]/edit
- 기존 데이터 pre-fill
- 상태 변경 (active ↔ inactive)
```

#### 2.8 숙소 검색/필터
```
Path: /properties?q=&location=&minPrice=&maxPrice=&guests=
구현:
- 키워드 검색 (title LIKE)
- 지역 필터 (셀렉트 박스)
- 가격 범위 필터 (슬라이더 또는 입력)
- 인원 필터
- URL 쿼리스트링 기반 (SSR 호환)
```

#### 2.9 로딩 스켈레톤
```
Files:
- src/app/loading.tsx
- src/app/properties/loading.tsx
- src/app/bookings/loading.tsx
- Tailwind animate-pulse 기반
```

#### 2.10 반응형 모바일 최적화
```
대상:
- Navbar 모바일 드롭다운 메뉴 완성
- 카드 레이아웃 1컬럼 전환
- 예약 폼 모바일 풀스크린
- 터치 친화적 버튼 크기 (min 44px)
```

#### 2.11 이미지 업로드
```
방식: 브라우저 → S3 Pre-signed URL 업로드 (또는 로컬 /public/uploads)
MVP 대안: input[type=file] + FormData → /api/upload → /public/uploads/ 저장
적용 대상:
- 숙소 등록 시 대표 이미지
- 리뷰 작성 시 이미지 첨부
```

#### 2.12 SEO 최적화
```
구현:
- 각 페이지 generateMetadata() 동적 메타데이터
- OG 태그 (숙소 상세: title + 이미지 + 가격)
- JSON-LD 구조화 데이터 (숙소, 리뷰)
- 시맨틱 HTML (header, main, section, article)
- sitemap.xml 자동 생성
```

### Sprint 3: 운영 안정성 (P2)

#### 2.13 이메일 알림
```
라이브러리: Resend (무료 100건/일)
알림 유형:
- 예약 접수 → 호스트에게
- 예약 확정 → 게스트에게
- 리뷰 작성 → 호스트에게
- 호스트 답글 → 게스트에게
```

#### 2.14 Rate Limiting
```
라이브러리: @upstash/ratelimit (Upstash Redis) 또는 in-memory
적용:
- POST /api/auth/register: 5회/분
- POST /api/reviews: 3회/분
- POST /api/bookings: 5회/분
```

#### 2.15 관리자 기능
```
Path: /admin
구현:
- 전체 숙소 관리 (승인/비활성화)
- 사용자 관리 (역할 변경, 차단)
- 예약 현황 대시보드
- 크롤링 숙소 검토 (source: "uploaded" 필터)
```

#### 2.16 에러 모니터링
```
옵션: Sentry (무료 5K 이벤트/월)
구현:
- Next.js Sentry 플러그인 설치
- API 에러 자동 캡처
- 클라이언트 에러 바운더리 연동
```

## 3. 기술 결정 사항

### DB: SQLite 유지
- **장점**: 배포 단순 (서버리스 불가하지만 VPS/Docker 가능), 설정 불필요
- **유의점**: 동시 쓰기 제한 → WAL 모드 활성화 권장
- **마이그레이션**: `PRAGMA journal_mode=WAL;` 적용
- **백업**: cron으로 dev.db 주기적 복사

### 배포 전략
```
옵션 1: Docker (권장)
- Dockerfile: Node.js + SQLite 포함
- Volume: dev.db 영속 저장
- Railway / Fly.io / DigitalOcean App Platform

옵션 2: VPS (Lightsail / Vultr)
- PM2 프로세스 매니저
- Nginx 리버스 프록시
- Let's Encrypt SSL
```

### 이미지 저장
```
MVP: /public/uploads/ 로컬 저장 (SQLite와 동일 서버)
확장: Cloudflare R2 (S3 호환, 무료 10GB)
```
