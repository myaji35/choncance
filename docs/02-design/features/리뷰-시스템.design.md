## 1. API 스펙 (API Specification)

### 1-1. 리뷰 작성
```
POST /api/reviews

Request Body:
{
  bookingId: string,
  rating: number,     // 1-5
  content: string     // 10-500자
}

Response 201:
{
  id: string,
  bookingId: string,
  propertyId: string,
  rating: number,
  content: string,
  createdAt: string
}

Errors:
  400 - 유효성 검사 실패 (별점 범위, 텍스트 길이)
  401 - 미인증
  403 - 본인 예약이 아님
  409 - 이미 리뷰 작성됨
  422 - 체크아웃 미완료 예약
```

### 1-2. 숙소 리뷰 목록 조회
```
GET /api/properties/[id]/reviews?page=1&limit=10

Response 200:
{
  reviews: [
    {
      id: string,
      rating: number,
      content: string,
      guestName: string,     // 마스킹됨: "홍*동"
      createdAt: string,
      hostReply: string | null,
      repliedAt: string | null
    }
  ],
  pagination: {
    total: number,
    page: number,
    limit: number,
    hasMore: boolean
  },
  summary: {
    avgRating: number,       // 소수점 1자리
    totalCount: number
  }
}
```

### 1-3. 호스트 답글 등록
```
POST /api/reviews/[id]/reply

Request Body:
{
  content: string    // 10-300자
}

Response 200:
{
  reviewId: string,
  hostReply: string,
  repliedAt: string
}

Errors:
  400 - 유효성 검사 실패
  401 - 미인증
  403 - 본인 숙소 리뷰가 아님
  409 - 이미 답글 작성됨
```

### 1-4. 호스트 리뷰 관리 목록
```
GET /api/host/reviews?page=1&limit=20

Response 200:
{
  reviews: [
    {
      id: string,
      rating: number,
      content: string,
      guestName: string,
      createdAt: string,
      hasReply: boolean,
      hostReply: string | null,
      property: { id: string, name: string }
    }
  ],
  pagination: { total, page, limit, hasMore }
}
```

## 4. 데이터 플로우 (Data Flow)

### 리뷰 작성 플로우
```
게스트 → /bookings/[id] 페이지
  → "리뷰 작성" 버튼 클릭 (조건: status=COMPLETED, checkOut < now, review=null)
  → /bookings/[id]/review 페이지
  → 별점 + 텍스트 입력
  → POST /api/reviews
    → Auth 확인 (Clerk)
    → 예약 소유권 확인
    → 체크아웃 날짜 확인
    → 중복 리뷰 확인 (bookingId unique)
    → prisma.review.create()
    → 성공 토스트 "리뷰가 등록되었습니다"
    → /bookings/[id] 리다이렉트
```

### 호스트 답글 플로우
```
호스트 → /host/reviews 페이지
  → 미답변 리뷰 확인
  → "답글 작성" 버튼 클릭
  → HostResponseForm 인라인 표시
  → 텍스트 입력 후 등록
  → POST /api/reviews/[id]/reply
    → Auth 확인 (호스트 권한)
    → 리뷰 소유권 확인 (본인 숙소)
    → 중복 답글 확인 (hostReply not null)
    → prisma.review.update({ hostReply, repliedAt })
    → UI 인라인 업데이트
```

## 5. 유틸리티 함수

```typescript
// src/lib/utils/review.ts

// 이름 마스킹
export function maskName(name: string): string {
  if (!name || name.length < 2) return name
  if (name.length === 2) return name[0] + '*'
  return name[0] + '*' + name[name.length - 1]
}
// "홍길동" → "홍*동", "김철수" → "김*수"

// 평균 별점 포맷
export function formatRating(avg: number | null): string {
  if (!avg) return '0.0'
  return avg.toFixed(1)
}

// 리뷰 작성 가능 여부
export function canWriteReview(booking: {
  status: string
  checkOut: Date
  review: unknown | null
}): boolean {
  return (
    booking.status === 'COMPLETED' &&
    new Date(booking.checkOut) < new Date() &&
    booking.review === null
  )
}
```

## 6. Zod 스키마

```typescript
// src/lib/validations/review.ts
import { z } from 'zod'

export const reviewSchema = z.object({
  bookingId: z.string().cuid(),
  rating: z.number().int().min(1, '별점을 선택해주세요').max(5),
  content: z
    .string()
    .min(10, '최소 10자 이상 입력해주세요')
    .max(500, '최대 500자까지 입력 가능합니다'),
})

export const hostReplySchema = z.object({
  content: z
    .string()
    .min(10, '최소 10자 이상 입력해주세요')
    .max(300, '최대 300자까지 입력 가능합니다'),
})
```
