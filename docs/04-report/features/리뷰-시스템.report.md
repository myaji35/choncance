## 3. 완성된 기능

### 3.1 API Routes (3개)

| 엔드포인트 | 파일 | 기능 |
|-----------|------|------|
| `POST /api/reviews` | `api/reviews/route.ts` | 리뷰 작성 + SNS 크레딧 트랜잭션 |
| `GET /api/reviews` | `api/reviews/route.ts` | 리뷰 목록 (propertyId 필터, 페이지네이션, 마스킹) |
| `POST /api/reviews/[id]/reply` | `api/reviews/[id]/reply/route.ts` | 호스트 답글 등록 |
| `GET /api/properties/[id]/reviews` | `api/properties/[id]/reviews/route.ts` | 숙소별 리뷰 (마스킹, 페이지네이션, summary) |

### 3.2 컴포넌트 (5개)

| 컴포넌트 | 파일 | 특징 |
|---------|------|------|
| **StarRating** | `components/review/StarRating.tsx` | 독립 컴포넌트, 인터랙티브/읽기전용 모드, 3가지 사이즈 |
| **ReviewCard** | `components/review/ReviewCard.tsx` | SLDS 스타일, 호스트 답글 섹션, isHost/onReply |
| **ReviewList** | `components/review/ReviewList.tsx` | SSR 초기 데이터 + "더 보기" 클라이언트 페이지네이션 |
| **ReviewForm** | `components/review/ReviewForm.tsx` | react-hook-form + Zod, SNS 동의 체크박스 |
| **HostResponseForm** | `components/review/HostResponseForm.tsx` | 10-300자, 취소 기능 |

### 3.3 페이지 (4개)

| 페이지 | 경로 | 패턴 |
|--------|------|------|
| 리뷰 작성 | `/bookings/[id]/review` | Server Component + ReviewPageClient |
| 예약 상세 | `/bookings/[id]` | Server Component, 조건부 리뷰 버튼 |
| 숙소 상세 | `/property/[id]` | Server Component + ReviewList |
| 호스트 리뷰 관리 | `/host/reviews` | Server Component + HostReviewsClient |

### 3.4 유틸리티

| 파일 | 함수 |
|------|------|
| `lib/utils/review.ts` | `maskName()`, `formatRating()`, `canWriteReview()` |
| `lib/validations/review.ts` | `createReviewSchema`, `hostReplySchema` |
| `lib/auth.ts` | `getCurrentUser()`, `requireAuth()` |
| `lib/prisma.ts` | Prisma 7 + libsql adapter singleton |

### 3.5 데이터 모델

| 모델 | 주요 필드 |
|------|----------|
| **Review** | rating(1-5), content, images(JSON), snsShareConsent, creditAwarded, hostReply, repliedAt |
| User | email, name, role(GUEST/HOST/ADMIN) |
| Property | title, location, status(draft/active), source, hostId |
| Booking | status(PENDING/CONFIRMED/COMPLETED/CANCELLED), checkIn/Out |
| CreditHistory | type(EARNED_REVIEW_SNS), amount, userId |
