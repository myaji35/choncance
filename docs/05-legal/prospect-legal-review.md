**작성일**: 2026-04-07
**대상 전략**: VINTEE의 공공데이터 기반 펜션 사전 등록 + GraphRAG 추천 + Claim 플로우
**용도**: 외부 변호사 자문 질의서 + 자체 방어선 점검
**요청자**: Gagahoho, Inc. (VINTEE)

## 1. 전략 요약 (변호사에게 전달)

VINTEE는 한국 촌캉스(농촌 휴가) 숙소 예약 플랫폼입니다. 초기 콜드 스타트 문제를 해결하고 AI 검색(ChatGPT·Perplexity·Google AI Overview) 시대에 한국 촌캉스 카테고리를 선점하기 위해 다음 전략을 검토 중입니다.

### 1.1 핵심 흐름

1. **공공데이터 수집**: 한국관광공사 TourAPI, 공공데이터포털, 지자체 농촌관광 DB 등 공공누리 1유형/CC BY 라이선스 데이터만 수집
2. **자동 구조화 데이터 생성**: Schema.org LodgingBusiness / FAQPage JSON-LD, 시맨틱 HTML, sitemap, llms.txt 자동 생성
3. **GraphRAG 태그 추출**: OpenAI GPT-4o-mini로 숙소 설명 → 태그(테마·분위기·활동·대상) 자동 생성, SQLite에 저장
4. **Prospect 상태 노출**: 수집한 숙소를 `status: "prospect"`로 사이트에 노출. 단, 예약 불가 배너 + "호스트이신가요? 인수하기" CTA 의무 표시
5. **Claim 플로우**: 호스트가 사업자등록증 또는 전화번호 인증으로 인수 → 3개월 무료 트라이얼
6. **구독 + 수수료 모델**:
   - Starter 월 1만원 + 수수료 5% (No Sale, No Fee 가드레일)
   - Pro 월 3만원 + 수수료 2.5%
   - Success Fee Only 월 0원 + 수수료 8%

### 1.2 경쟁사 포지셔닝

에어비앤비·야놀자는 호스트가 직접 등록하는 플랫폼. VINTEE는 "AI 시대의 촌캉스 인텔리전스 엔진"으로 차별화. 호스트에게 "가입 안 해도 이미 당신 숙소가 ChatGPT에 노출되고 있다"는 가치를 선제 제공.

### 1.3 유사 선례

- **Google 비즈니스 프로필(Google Business Profile)**: 상호·주소·전화번호 등 공공 정보로 사업자 페이지 자동 생성 → 사업자가 "Claim" 후 관리
- **TripAdvisor**: 숙소·레스토랑을 사용자 제보로 사전 등록 → 사업자가 Claim → 유료 옵션
- **네이버 플레이스**: 공공 정보로 업체 사전 등록 → 사업자 인증 후 관리 가능
- VINTEE 전략은 위 선례의 **핵심 플로우를 답습**하면서, GraphRAG 추천 + 저렴한 수수료로 차별화

## 2. 법적 쟁점 및 질의사항

### 쟁점 1: 공공데이터 수집과 사용 범위

**질문 1-1**
한국관광공사 TourAPI(공공누리 제1유형: 출처 표시)로 제공되는 펜션의 상호·주소·전화번호·설명·사진을 VINTEE 웹사이트에 게시하는 것이 합법입니까? 상업적 이용이 포함됩니까?

**질문 1-2**
공공데이터포털의 "숙박업 영업 신고 현황" 데이터(CC BY)에서 얻은 사업자명·주소·전화를 Property 테이블에 저장하고 검색/추천 엔진 학습에 활용해도 되는지 확인 요청합니다.

**질문 1-3**
공공데이터를 원본 그대로 게시하지 않고 LLM(GPT-4o-mini)으로 요약·재구성한 텍스트, 자동 생성한 FAQ, 태그(예: "한옥/가족/가을")를 게시하는 것은 저작권·2차 저작물 관점에서 문제가 있습니까?

**질문 1-4**
숙소 사진의 경우:
- (a) TourAPI에서 제공하는 공공 사진을 원본 링크로 `<img src>` 렌더링하는 경우
- (b) 사진을 다운받아 VINTEE 서버에 재호스팅하는 경우
- (c) 사진 없이 placeholder 이미지(자체 생성)로 대체하는 경우
각각의 리스크와 권장 접근 방식을 알려주십시오.

### 쟁점 2: 호스트 동의 없는 상호·사업자 정보 게시

**질문 2-1**
호스트(개인사업자)의 상호·주소·전화번호는 「개인정보보호법」상 개인정보에 해당합니까? 사업자등록 공개 정보는 개인정보 예외에 해당합니까? (판례 또는 가이드라인 인용 요청)

**질문 2-2**
"이 숙소는 공공데이터 기반으로 자동 등록되었습니다. VINTEE를 통한 예약은 불가하며, 본인이 호스트이시라면 [인수하기] 버튼으로 직접 관리하실 수 있습니다. 정보 삭제 요청은 contact@vintee.kr"이라는 고지를 페이지 최상단에 고정 노출한다면 동의 없는 게시의 위법성이 완화됩니까?

**질문 2-3**
호스트의 "삭제 요청(Opt-out)"은 즉시 처리(24시간 이내)하고, 관련 절차를 이용약관·개인정보처리방침에 명시할 계획입니다. 이 Opt-out 메커니즘이 「개인정보보호법」 제37조(개인정보의 처리 정지 요구권) 대응으로 충분합니까?

**질문 2-4**
호스트 본인이 수집된 정보가 **틀렸다**고 주장할 경우의 정정 프로세스는? 관리자가 1영업일 이내에 정정 가능한 구조를 준비 중입니다.

### 쟁점 3: 부정경쟁방지법·표시광고법

**질문 3-1**
Prospect 상태의 숙소 페이지 상단에 "예약 불가" 노란색 배너 + `<script type="application/ld+json">` 내 `"availability": "https://schema.org/OutOfStock"` 명시 + 예약 버튼 비활성(회색) 처리를 할 경우, 소비자가 "예약 가능한 것으로 오인"할 여지가 법적으로 제거됩니까?

**질문 3-2**
VINTEE가 해당 숙소의 실제 호스트가 아님에도 사이트에 상호를 게시하는 것이 「부정경쟁방지법」제2조 1호 가목(상품표지 혼동 유발)에 해당할 가능성은? 위 배너·고지 조치로 방어 가능합니까?

**질문 3-3**
"이 숙소의 호스트이시면 인수하세요"라는 CTA가 "VINTEE가 그 숙소를 운영한다"는 오해를 유발하지 않도록 보호하는 카피 표현을 권장해 주십시오.

### 쟁점 4: 업무방해죄 및 경쟁사 관계

**질문 4-1**
에어비앤비·야놀자 등에 이미 등록된 호스트의 숙소가 VINTEE에 prospect로 자동 노출될 경우, 해당 호스트가 VINTEE를 상대로 「업무방해죄」(형법 제314조) 또는 민사 손해배상 청구를 할 법적 근거가 생깁니까?

**질문 4-2**
에어비앤비·야놀자·여기어때 등의 플랫폼에서 **절대 크롤링하지 않고** 오직 공공데이터만 사용한다는 조건에서, 경쟁사가 VINTEE를 "데이터 부정이용"으로 제소할 수 있는 여지가 있습니까?

**질문 4-3**
2016년 잡코리아 vs 사람인, 2020년 야놀자 vs 여기어때, 2022년 네이버 부동산 매물 수집 사건 등 선례를 고려할 때 VINTEE 전략이 유사 리스크에 노출될 가능성은? 위 사건과의 핵심 차이점(공공데이터 전용 사용)이 방어 논거로 유효합니까?

### 쟁점 5: Claim(인수) 플로우의 법적 유효성

**질문 5-1**
호스트가 Claim을 통해 숙소 관리권을 인수할 때 본인 확인 방법으로 다음을 고려하고 있습니다. 각 방법의 법적 유효성 및 권장 순위를 알려주십시오:
- (a) 사업자등록증 이미지 업로드 + 사업자 조회 API 대조
- (b) 해당 숙소 전화번호로 인증번호 SMS 발송
- (c) 이메일 도메인 확인
- (d) 지자체 인허가 DB 조회

**질문 5-2**
Claim 성공 후 해당 숙소의 데이터(사진·설명·태그 등)에 대한 저작권/관리권은 누구에게 귀속됩니까? 이용약관에 어떻게 명시해야 합니까?

**질문 5-3**
한 숙소에 대해 여러 명이 Claim을 신청한 경우(예: 전 호스트 vs 신규 호스트, 대표자 vs 직원) 분쟁 해결 절차를 어떻게 설계해야 합니까?

**질문 5-4**
Claim한 호스트가 나중에 이용약관을 위반하거나 결제 실패로 `dormant` 상태가 되면, 해당 숙소를 다시 `prospect`로 되돌리는 것이 가능합니까? 아니면 "삭제 요청"만 가능합니까?

### 쟁점 6: 구독·수수료 모델

**질문 6-1**
월 구독(1만원/3만원) + 예약 수수료(2.5%/5%/8%) 하이브리드 모델에서 "전자상거래법"상 통신판매중개자로서 의무는? 약관 필수 기재사항, 환불 정책, 분쟁 조정 의무 등을 명시해 주십시오.

**질문 6-2**
"No Sale, No Fee"(월 예약 0건 시 구독료 자동 면제) 정책이 전자상거래법·표시광고법 관점에서 문제가 있습니까? 정기결제 표시 의무와의 관계는?

**질문 6-3**
"3개월 무료 트라이얼" 후 자동 유료 전환 시 다크패턴 규제(2024년 개정 전자상거래법) 준수 방안을 알려주십시오.

**질문 6-4**
수수료 5%/2.5%는 PG사(토스페이먼츠 등) 수수료 2.5-3.3% 별도 부담을 가정합니다. 게스트에게 "결제 수수료 별도 표시" 방식이 적법한지, 아니면 내부 흡수해야 하는지 검토 요청.

## 3. 자체 방어선 (코드/UX 구현 예정)

### 3.1 UX 방어선

| 방어선 | 위치 | 구현 |
|---|---|---|
| "예약 불가" 노란 배너 | Prospect 페이지 최상단 고정 | `UnclaimedBanner.tsx` |
| "호스트이신가요? 인수" CTA | 예약 버튼 자리 대체 | 예약 폼 영역 |
| 정보 삭제 요청 채널 | 모든 Prospect 페이지 하단 | `contact@vintee.kr` + 전용 폼 |
| 출처 표기 | 모든 Prospect 데이터 | "출처: 한국관광공사 TourAPI" 링크 |
| "AI 자동 생성" 배지 | LLM 생성 콘텐츠 | FAQ/요약 섹션 우측 상단 |
| `availability: OutOfStock` | JSON-LD | `PropertyJsonLd.tsx` 확장 |
| 회색 예약 버튼 (비활성) | 예약 영역 | disabled + tooltip |

### 3.2 데이터 방어선

| 방어선 | 구현 |
|---|---|
| 공공데이터 소스 기록 | `Property.dataSource`, `Property.sourceUrl` |
| 수집 시각 기록 | `Property.collectedAt` |
| 호스트 Opt-out 플래그 | `Property.visible: false` |
| 관리자 차단 | `Property.status: "suspended"` |
| Claim 추적 | `Property.claimStatus`, `claimedBy`, `claimedAt` |
| LLM 생성 표기 | `PropertyTag.source: "auto_llm"` |

### 3.3 약관 방어선

| 방어선 | 약관 조항 |
|---|---|
| 호스트 권리 | "호스트는 언제든 Opt-out 요청 가능, 24시간 이내 처리" |
| Claim 저작권 | "Claim 후 콘텐츠 저작권은 호스트에게 귀속, VINTEE는 서비스 제공을 위한 사용권만 보유" |
| LLM 생성 책임 | "자동 생성 콘텐츠는 참고용이며 호스트가 정정 요청 가능" |
| 분쟁 해결 | "관할 법원: 서울중앙지방법원" |
| 환불 정책 | "구독료 월 단위 일할 환불" |

## 4. 변호사 자문 진행 절차

### 4.1 자문 목적
1. 위 **쟁점 1~8 질의사항**에 대한 법률 의견서 확보
2. **이용약관 v2 초안** 작성·검토
3. **개인정보처리방침 v2 초안** 작성·검토
4. Claim 플로우 본인 확인 절차 법적 유효성 검증
5. 분쟁 시 대응 플레이북 수립

### 4.2 자문 비용 예산
- 1회 대면/원격 자문: 30~50만원
- 약관/처리방침 검토: 추가 20~50만원
- 총 예산: **50~100만원**

### 4.3 자문 후 Go/No-Go 기준

| 결과 | 조치 |
